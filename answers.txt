1. When I type a website address and press Enter, my browser asks the DNS for the website's IP address. The request goes to the server, which sends the webpage back to my browser.

2.A client requests information, while a server provides it.
Client: Google Chrome.
Server: Google's web server.
3. DNS stands for Domain Name System. It changes website names like google.com into IP addresses that computers can understand. Without DNS, we would have to remember long numbers instead of simple website names.
4.A firewall and a load balancer are both part of how the web works. What does each one do? How are they different?
5. A web server mainly delivers web pages, images, and other files to users. An application server handles the business logic of the application, such as processing user requests and communicating with the database. Backend developers work more with the application server because it is responsible for how the application functions behind the scenes.
6.The client does not connect directly to the database because it would expose sensitive data and create security risks. The application server checks requests, verifies users, and controls what information can be accessed. This keeps the system more secure and organized.
7. A university department page is more likely to be a static website because the information does not change very often. An MTN MoMo account page is dynamic because it displays personal account information like my balance and transaction history, which changes regularly.
8.Frontend is what users see, while backend handles the server and database. The backend checks passwords because it is more secure.
9.Backend developers use the command line to manage files, run programs, and work with servers more efficiently.
10.An absolute path shows the full file location.
Example: C:\Users\Bernice\answers.txt
A relative path shows the location from the current folder.
Example: answers.txt
11.The command line is faster, works on remote servers, and is an important skill for backend developers.
                         WEB REQUEST JOURNEY

Request →
+---------+
| Client  |
| Sends an HTTP request to access a website or application. |
+---------+
     |
     v
+---------+
| DNS     |
| Translates the domain name into the server's IP address. |
+---------+
     |
     v
+------------+
| Firewall   |
| Filters and blocks unauthorized or malicious traffic. |
+------------+
     |
     v
+---------------+
| Load Balancer |
| Distributes incoming requests across available servers. |
+---------------+
     |
     v
+-------------+
| Web Server  |
| Receives the request and serves static content or forwards it. |
+-------------+
     |
     v
+-------------------+
| Application Server|
| Processes business logic and handles dynamic requests. |
+-------------------+
     |
     v
+-----------+
| Database  |
| Stores and retrieves the requested data. |
+-----------+

Response ←
+-----------+
| Database  |
| Returns the requested data. |
+-----------+
     ^
     |
+-------------------+
| Application Server|
| Processes the data and prepares the response. |
+-------------------+
     ^
     |
+-------------+
| Web Server  |
| Sends the response back to the client. |
+-------------+
     ^
     |
+---------------+
| Load Balancer |
| Routes the response through the correct path. |
+---------------+
     ^
     |
+------------+
| Firewall   |
| Allows only safe outbound traffic to pass. |
+------------+
     ^
     |
+---------+
| Client  |
| Receives and displays the requested web page or data. |
+---------+
9.Backend developers need the command line to run applications, manage files, install packages, use Git, work with databases, and deploy applications. It is an essential tool that helps them work efficiently and manage servers, especially in Linux environments.
10.An absolute path is the full location of a file or folder from the root directory.
Example: C:\Users\Bernice\Documents\project\index.html

A relative path is the location of a file or folder based on the current directory.
Example: Documents\project\index.html
11.File Explorer and Finder are useful for basic tasks, but the CLI is faster, more efficient, and required for many backend development tasks, such as managing files, running programs, and working on servers.
12.~ means the user's home directory, while $ indicates the terminal is ready for you to enter a command.
